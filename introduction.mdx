## What is spec-loop?

spec-loop is a CLI tool that turns feature specifications into working code through autonomous development loops. It orchestrates Claude Code to:

1. Break down feature specs into individual tasks
2. Build each task using Claude Code
3. Review the implementation against your project's standards
4. Automatically fix issues and retry until it passes
5. Move to the next task

All while you focus on higher-level work or take a break.

## How It Works

<Steps>
  <Step title="Create a Feature Spec">
    Use the `/spec-loop-spec` Claude Code skill to create a feature spec with individual task files in `.agents/specs/`
    
    ```bash
    /spec-loop-spec Add user authentication with JWT tokens
    ```
  </Step>
  
  <Step title="Run the Loop">
    Launch spec-loop and watch it work through your tasks
    
    ```bash
    spec-loop run
    ```
    
    For each task, spec-loop:
    - **Build**: Claude Code implements the task
    - **Review**: Automated review checks against your project's standards
    - **Fix**: If issues found, Claude Code fixes them (up to 3 attempts)
    - **Commit**: Task marked done, moves to next one
  </Step>
  
  <Step title="Check Progress">
    Monitor status at any time
    
    ```bash
    spec-loop status
    ```
    
    Or use the `/spec-loop-status` skill in Claude Code
  </Step>
</Steps>

## Why spec-loop?

**Traditional development loop:**
- You write code
- You manually test
- You fix issues
- You repeat
- Hours or days per feature

**With spec-loop:**
- You write a spec (5-10 minutes)
- spec-loop builds, reviews, and fixes automatically
- You review the final result
- Ship features faster with consistent quality

## Core Concepts

### Specs and Tasks

A **spec** is a feature broken down into individual **tasks**. Each task is a focused unit of work (add a route, create a component, write tests).

```
.agents/specs/user-auth/
├── spec.md              # Feature overview
├── task-01.md           # Setup JWT config
├── task-02.md           # Create auth middleware
├── task-03.md           # Add login endpoint
└── ...
```

### AGENTS.md - Your Project's Rulebook

This file tells spec-loop (and Claude Code) how your project works:
- Architecture patterns
- Naming conventions
- Code style preferences
- Review criteria
- What to avoid

spec-loop reads this file during build and review phases to ensure consistency.

### Build-Review-Fix Loop

Each task goes through:

1. **Build phase**: Claude Code implements the task
2. **Review phase**: Automated review generates findings (must-fix vs should-fix)
3. **Fix phase** (if needed): Claude Code addresses must-fix issues
4. **Recheck**: Review runs again
5. **Done**: Task marked complete, or left in-review after max attempts

### Circuit Breaker

If spec-loop detects no progress after N iterations (default: 3), it opens the circuit breaker and stops. This prevents:
- Infinite loops on blocked tasks
- Wasting API credits
- Thrashing on unsolvable problems

You can resume after investigating the issue.
