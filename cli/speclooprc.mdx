## Overview

The `.speclooprc` file configures spec-loop behavior for your project. Created by `spec-loop init`, it uses a simple `KEY="value"` format.

## Location

```
project-root/.speclooprc
```

spec-loop looks for `.speclooprc` in the current directory when running commands.

## Format

```text
# Comments start with #
KEY="value"
ANOTHER_KEY="another value"

# Quotes are optional for simple values
NUMBER=42
```

## Configuration Options

### Project Settings

<ParamField path="PROJECT_TYPE" type="string">
  Project type identifier.
  
  **Auto-detected**: Based on project files (see [spec-loop init](/cli/init#project-type-detection))
  
  **Options**: `typescript`, `javascript`, `python`, `rust`, `go`, `ruby`, `java`, `swift`, `generic`
  
  **Example**: `PROJECT_TYPE="typescript"`
</ParamField>

<ParamField path="VERIFY_COMMAND" type="string" required>
  Command to verify code quality (linting, type checking).
  
  Runs during build phase after implementation.
  
  **Example**: `VERIFY_COMMAND="npm run lint && npm run typecheck"`
</ParamField>

<ParamField path="TEST_COMMAND" type="string" required>
  Command to run tests.
  
  Runs during review phase to validate functionality.
  
  **Example**: `TEST_COMMAND="npm test"`
</ParamField>

### Claude Code Settings

<ParamField path="CLAUDE_BIN" type="string">
  Path or name of Claude Code CLI binary.
  
  **Default**: `claude`
  
  **Override**: `CLAUDE_BIN` environment variable
  
  **Example**: `CLAUDE_BIN="/usr/local/bin/claude"`
</ParamField>

<ParamField path="CLAUDE_MODEL" type="string">
  Claude model to use for all phases.
  
  **Default**: Uses Claude Code's default model
  
  **Example**: `CLAUDE_MODEL="claude-opus-4"`
</ParamField>

<ParamField path="CLAUDE_EXTRA_ARGS" type="string">
  Additional arguments to pass to Claude Code CLI.
  
  **Override**: `CLAUDE_EXTRA_ARGS` environment variable
  
  **Example**: `CLAUDE_EXTRA_ARGS="--debug"`
</ParamField>

### Loop Limits

<ParamField path="MAX_LOOPS" type="number">
  Maximum number of build-review-fix iterations per run.
  
  **Default**: `25`
  
  **Override**: `SPECLOOP_MAX_LOOPS` environment variable or `--max-loops` CLI flag
  
  **Example**: `MAX_LOOPS=50`
</ParamField>

<ParamField path="MAX_REVIEW_FIX_LOOPS" type="number">
  Maximum review-fix retry attempts per task.
  
  **Default**: `3`
  
  **Override**: `SPECLOOP_MAX_REVIEW_FIX_LOOPS` environment variable or `--max-review-fix-loops` CLI flag
  
  **Example**: `MAX_REVIEW_FIX_LOOPS=5`
</ParamField>

<ParamField path="MAX_TASKS_PER_RUN" type="number">
  Maximum tasks to complete in a single run.
  
  **Default**: `0` (unlimited)
  
  **Override**: `SPECLOOP_MAX_TASKS_PER_RUN` environment variable or `--max-tasks` CLI flag
  
  Set to `0` for unlimited tasks.
  
  **Example**: `MAX_TASKS_PER_RUN=3`
</ParamField>

### Safety Settings

<ParamField path="CB_NO_PROGRESS_THRESHOLD" type="number">
  Number of consecutive iterations without progress before circuit breaker trips.
  
  **Default**: `3`
  
  Progress is defined as: task status changes or remaining task count decreases.
  
  **Example**: `CB_NO_PROGRESS_THRESHOLD=5`
</ParamField>

<ParamField path="CB_COOLDOWN_MINUTES" type="number">
  Minutes to wait before allowing retry after circuit breaker trips.
  
  **Default**: `30`
  
  **Example**: `CB_COOLDOWN_MINUTES=60`
</ParamField>

### Path Settings

<ParamField path="SPECS_DIR" type="string">
  Directory containing feature specs.
  
  **Default**: `.agents/specs`
  
  **Override**: `SPECLOOP_SPECS_DIR` environment variable
  
  **Example**: `SPECS_DIR="specs"`
</ParamField>

<ParamField path="SESSION_DIR" type="string">
  Directory for session logs.
  
  **Default**: `.spec-loop/sessions`
  
  **Override**: `SPECLOOP_SESSION_DIR` environment variable
  
  **Example**: `SESSION_DIR=".logs/spec-loop"`
</ParamField>

## Example Configuration

### TypeScript Project

```bash
# spec-loop configuration
# Generated by spec-loop init

# Project
PROJECT_TYPE="typescript"
VERIFY_COMMAND="npm run lint && npm run typecheck"
TEST_COMMAND="npm test"

# Claude Code
# CLAUDE_MODEL=""

# Loop limits
# MAX_LOOPS=25
# MAX_REVIEW_FIX_LOOPS=3
# MAX_TASKS_PER_RUN=0

# Safety
# CB_NO_PROGRESS_THRESHOLD=3
# CB_COOLDOWN_MINUTES=30

# Paths
# SPECS_DIR=".agents/specs"
# SESSION_DIR=".spec-loop/sessions"
```

### Rust Project

```bash
# Project
PROJECT_TYPE="rust"
VERIFY_COMMAND="cargo clippy && cargo check"
TEST_COMMAND="cargo test"

# Increase limits for complex project
MAX_LOOPS=50
MAX_REVIEW_FIX_LOOPS=5
```

### Python Project with Custom Paths

```bash
# Project
PROJECT_TYPE="python"
VERIFY_COMMAND="ruff check . && mypy ."
TEST_COMMAND="pytest"

# Custom paths
SPECS_DIR="docs/specs"
SESSION_DIR=".logs/spec-loop"
```

## Configuration Precedence

Settings are applied in this order (later overrides earlier):

1. **Default values** (hardcoded in spec-loop)
2. **`.speclooprc`** file
3. **Environment variables** (see [Environment Variables](/cli/environment-variables))
4. **CLI flags** (see [spec-loop run](/cli/run))

### Example Override Chain

```text
# .speclooprc
MAX_LOOPS=25

# Environment variable overrides .speclooprc
export SPECLOOP_MAX_LOOPS=50

# CLI flag overrides both
spec-loop run --max-loops 10
# Result: Uses 10 loops
```

## Validation

spec-loop validates configuration on startup:

<ResponseField name="Missing .speclooprc">
  ```text
  ✗ No .speclooprc found. Run 'spec-loop init' first.
  ```
  
  **Solution**: Run `spec-loop init`
</ResponseField>

<ResponseField name="Invalid MAX_LOOPS">
  ```text
  ✗ MAX_LOOPS must be a positive integer (got: 0)
  ```
  
  **Solution**: Set `MAX_LOOPS` to a value > 0
</ResponseField>

<ResponseField name="Invalid MAX_REVIEW_FIX_LOOPS">
  ```text
  ✗ MAX_REVIEW_FIX_LOOPS must be a positive integer (got: 0)
  ```
  
  **Solution**: Set `MAX_REVIEW_FIX_LOOPS` to a value > 0
</ResponseField>

## Comments

Lines starting with `#` are ignored:

```text
# This is a comment
PROJECT_TYPE="typescript"  # Inline comments NOT supported

# Temporarily disable a setting
# MAX_TASKS_PER_RUN=3
```

## Editing

### Manual editing

```bash
# Edit with your preferred editor
vim .speclooprc
nano .speclooprc
code .speclooprc
```

### Reinitialize

```bash
# Recreate with auto-detection
spec-loop init --force

# Recreate with custom commands
spec-loop init --force --verify-cmd "make lint" --test-cmd "make test"
```

## Best Practices

<Tip>
**Keep commented defaults** - The generated `.speclooprc` includes commented default values for reference.
</Tip>

<Tip>
**Commit to version control** - Share configuration across team:
```bash
git add .speclooprc
git commit -m "Add spec-loop configuration"
```
</Tip>

<Tip>
**Use environment variables for secrets** - Don't put API keys in `.speclooprc`. Use environment variables instead.
</Tip>

<Warning>
**Avoid circular dependencies** - Don't configure `VERIFY_COMMAND` to run `spec-loop run`.
</Warning>

## Troubleshooting

### Config not loading

```bash
# Check if file exists
ls -la .speclooprc

# Check current directory
pwd

# Verify file contents
cat .speclooprc
```

### Syntax errors

```bash
# Test configuration loading
spec-loop status
# If config is invalid, you'll see an error
```
